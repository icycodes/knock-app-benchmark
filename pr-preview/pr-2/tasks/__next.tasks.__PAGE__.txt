To enable this, the trigger request must include a `cancellation_key`, and a subsequent `POST /v1/workflows/{key}/cancel` request must reference the same `cancellation_key`.

The Knock workspace used for this task already has two channels configured:

- A Mailtrap email channel with channel key `mailtrap`.
